# Supplementary Figure 5: Comparison with Monte Carlo Tree Search (MCTS)

This directory contains the scripts and data required to reproduce 
**Supplementary Figure 5** of the paper *Containment of Escaping 
Targets in Complex Networks*.

Supplementary Figure 5 compares MobileIsolator against Monte Carlo 
Tree Search (MCTS) — a strong but expensive search-based baseline 
that performs many forward rollouts at decision time. The 
experiment shows that MobileIsolator approaches the decision 
quality of MCTS at orders-of-magnitude lower computational cost. 
To keep MCTS computationally tractable, all methods are evaluated 
on small networks ($N = 50$), with $20{,}000$ MCTS simulations 
per decision and progressive widening.


## Files

### Scripts

| Script | Role |
|--------|------|
| `mcts_final_new.py` | Self-contained module implementing the MCTS solver (`solve_with_improved_mcts`) used by the comparison script. No `__main__`; not run directly. |
| `comparison_tia_rl_mcts_final.py` | Generates the data: runs MobileIsolator, Adaptive TIA, and MCTS on the four synthetic topology–target regimes across three $\lambda$ values, averaged over $100$ independent simulations per condition. |
| `draw_mcts_comparison.py` | Renders Supplementary Figure 5 from the CSV. |

### Data (`data/`)

| File | Description | Source |
|------|-------------|--------|
| `results_mcts_n50_sim20000.csv` | Aggregate $P_C$ and ANC statistics (means and SEMs) and per-method wall-clock times for each $(\lambda, \textrm{regime}, \textrm{method})$ tuple. | Generated by `comparison_tia_rl_mcts_final.py` |


## Dependencies

`comparison_tia_rl_mcts_final.py` imports the MCTS solver from 
the local module:

```python
from mcts_final_new import solve_with_improved_mcts
```

and shared utilities from the Fig. 3 directory:

```python
from comparison_method import (...)              # ../../fig3/
```

The MobileIsolator agent class is loaded transitively from the 
repository-level `train/` directory:

```python
from ppo_gnn_moving_gat_dualstream_final import DualStreamAgent  # ../../train/
```

The pre-trained policy weights are loaded from:

```
../../train/gnn_ppo_dual_stream_n50-100__seed42__1769653775/model.pt
```

Path resolution is automatic; the script locates `fig3/` and 
`train/` relative to its own location (two levels up, since this 
directory is a sub-directory of `fig_sm/`). The plotting script 
`draw_mcts_comparison.py` has no such dependency — it only reads 
the CSV from `data/`.


## Reproducing the figure

The repository ships with the pre-computed CSV in `data/`, so the 
figure can be reproduced directly from the shipped data:

```bash
python draw_mcts_comparison.py
```

This will produce `mcts_comparison.pdf` and `.png` in this 
directory.

### Regenerating the data from scratch

MCTS is expensive: $20{,}000$ simulations per decision, across 
four regimes, three $\lambda$ values, and $100$ episodes per 
condition. Expect **roughly $30$–$50$ hours** on a typical 
workstation using parallel workers.

```bash
python comparison_tia_rl_mcts_final.py
```

This produces `data/results_mcts_n50_sim20000.csv`. Once the CSV 
is in place:

```bash
python draw_mcts_comparison.py
```


## Configuration

Key parameters can be adjusted inside the `Config` class of 
`comparison_tia_rl_mcts_final.py`:

- `N_NODES = 50` — synthetic network size (small to keep MCTS 
  tractable)
- `TARGET_RATIO = 0.05`
- `LAMBDA_LEVELS = [1, 10, 20]` — the three $\lambda$ values 
  tested
- `SIMULATION_TIMES = 100` — independent simulations per condition
- `MCTS_SIMULATIONS = 20000` — MCTS rollouts per decision
- `TOP_K = 47` — progressive-widening candidate count, set to 
  approximately $N \cdot (1 - \textrm{target\_ratio})$
- `SUBTREE_REUSE`, `USE_DETERMINIZATION` — MCTS flags
- `N_WORKERS` — multiprocessing pool size. Automatically capped 
  at $60$ on Windows (due to the 63-handle limit of 
  `WaitForMultipleObjects`); set to $77$ on Linux / macOS.

For the rendering script, no configuration is normally needed; 
`CSV_PATH` and `SAVE_DIR` are resolved automatically.
