# Supplementary Figure 1: Validity of the evasion factor lambda

This directory contains the scripts and data required to reproduce 
**Supplementary Figure 1** of the paper *Containment of Escaping 
Targets in Complex Networks*.

Supplementary Figure 1 verifies the central modelling assumption 
that the evasion factor $\lambda \equiv p/r$ captures the 
essential difficulty of an episode: any two $(p, r)$ pairs that 
share the same $\lambda$ should yield statistically 
indistinguishable performance. The figure shows the containment 
cost $P_C$ and average normalized connectivity (ANC) of all four 
methods, as a function of $r$ at three fixed values of $\lambda 
\in \{1, 10, 20\}$ and across the four synthetic 
topology–target regimes. If the data-collapse hypothesis holds, 
each curve should be flat.


## Files

### Scripts

| Script | Role |
|--------|------|
| `lambda_validity.py` | Generates the data: for each of the three target $\lambda$ values, sweeps $r \in \{0.01, 0.02, \ldots, 0.05\}$, setting $p = \lambda \cdot r$ at each step (skipping pairs where $p \notin (0, 1]$). Each $(r, p)$ point is averaged over 500 independent simulations on the four regimes. |
| `plot_lambda_combined.py` | Renders the final 6 row $\times$ 4 column figure (one row pair per $\lambda$ block) from the CSV. |

### Data (`data/`)

| File | Description | Source |
|------|-------------|--------|
| `lambda_validity_results_500.csv` | Aggregate $P_C$ and ANC statistics (means, std, SEM over 500 simulations) for each $(\lambda, r, \textrm{regime}, \textrm{method})$ tuple. | Generated by `lambda_validity.py` |


## Dependencies

`lambda_validity.py` imports shared utilities from the Fig. 3 
directory:

```python
from comparison_method import (...)              # ../../fig3/
```

It also loads the MobileIsolator agent class transitively from 
the repository-level `train/` directory:

```python
from ppo_gnn_moving_gat_dualstream_final import DualStreamAgent  # ../../train/
```

The pre-trained policy weights are loaded from:

```
../../train/gnn_ppo_dual_stream_n50-100__seed42__1769653775/model.pt
```

Path resolution is automatic; the script locates `fig3/` and 
`train/` relative to its own location (two levels up, since this 
directory is a sub-directory of `fig_sm/`). The plotting script 
`plot_lambda_combined.py` has no such dependency — it only reads 
the CSV from `data/`.


## Reproducing the figure

The repository ships with the pre-computed CSV in `data/`, so the 
figure can be reproduced directly from the shipped data:

```bash
python plot_lambda_combined.py
```

This will produce `lambda_validity_combined.pdf` and `.png` in 
this directory.

### Regenerating the data from scratch

To regenerate the CSV from scratch, run:

```bash
python lambda_validity.py
```

This produces `data/lambda_validity_results_500.csv`. Once the 
CSV is in place:

```bash
python plot_lambda_combined.py
```


## Configuration

Key parameters can be adjusted at the top of each script:

- `lambda_validity.py`:
  - `LAMBDA_TARGETS = [1, 10, 20]` — the three $\lambda$ values 
    to test
  - `ATTACK_RATIO_VALUES` — the $r$ sweep grid (default 0.01–0.05)
  - `N_NODES = 1024`, `TARGET_RATIO = 0.05` — synthetic network 
    and target configuration
  - `SIMULATION_TIMES = 500` — independent simulations per condition
  - `N_WORKERS`, `N_WORKERS_RL` — multiprocessing pool sizes
- `plot_lambda_combined.py`:
  - `LAMBDA_TARGETS = [1, 10, 20]` — must match the values in the 
    CSV
  - `R_MIN`, `R_MAX`, `X_TICKS` — x-axis range and tick locations
