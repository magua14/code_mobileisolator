# Supplementary Tables 1–3: Ablation study on real-world scenarios

This directory contains the scripts, data, and ablation model 
weights required to reproduce **Supplementary Tables 1–3** of the 
paper *Containment of Escaping Targets in Complex Networks*.

Supplementary Tables 1–3 measure how each architectural component 
of MobileIsolator contributes to its performance, by comparing 
the full model against three ablation variants on all six 
real-world isolation scenarios at three evasion-factor levels 
($\lambda \in \{1, 10, 20\}$). Each table corresponds to a 
metric: Supplementary Table 1 reports the containment cost 
$P_C$, Supplementary Table 2 reports ANC, and Supplementary Table 
3 reports the scenario-specific containment metric (invasion 
ratio, exposure ratio, infiltration ratio, etc.).

The four model variants compared are:

| Variant | Architecture |
|---------|--------------|
| **Full** | Complete MobileIsolator (GAT local stream + global stream + virtual node + attention readout). |
| **No-Global** | Removes the global stream, keeping only local GAT message passing. |
| **No-Attn** | Replaces attention-based readout with mean pooling. |
| **No-VNode** | Removes the virtual node from the GAT backbone. |


## Files

### Scripts

| Script | Role |
|--------|------|
| `comparison_ablation_realworld.py` | Generates the data: evaluates all four model variants on the six real-world scenarios, across three $\lambda$ levels, averaged over $50$ independent simulations per condition. |
| `plot_ablation_table_realworld.py` | Renders Supplementary Tables 1–3 from the summary CSV. |

### Data (`data/`)

| File | Description | Source |
|------|-------------|--------|
| `summary_ablation_realworld.csv` | Per-(scenario, variant, $\lambda$) aggregate $P_C$ / ANC / scenario-specific-metric statistics (mean ± SEM). | Generated by `comparison_ablation_realworld.py` |
| `results_ablation_realworld.csv` | Long-format raw results, one row per simulation. | Generated by `comparison_ablation_realworld.py` |

## Dependencies

`comparison_ablation_realworld.py` imports across three 
sibling directories of this repository, with paths resolved 
relative to its own location (`fig_sm/sm_table1_3/`):

| Imports from | What it provides | Used for |
|--------------|------------------|----------|
| `../../train/` | `DualStreamAgent` | Loading the trained policy networks for all four variants |
| `../../fig3/` | `comparison_method.compute_params_from_lambda` | Translating $\lambda$ into the simulator parameters $(p, r)$ |
| `../../fig6/` | The six `draw_*_new.py` scenario modules (`draw_covid19_new`, `draw_city_flood_new`, `draw_invasive_species_new`, `draw_smuggling_new`, `draw_socialbot_new`, `draw_fugitive_chasing_new`) | Loading the per-scenario networks and selecting targets with the same logic used for Fig. 6 |

Each scenario's input network is read from the shared Fig. 6 data 
directory:

```
../../fig6/data/fig6_data/{scenario}_network.graphml
```

so no separate copy of the raw real-world networks is needed in 
this directory.

The plotting script `plot_ablation_table_realworld.py` has no 
such dependency — it only reads the CSV from `data/`.


## Reproducing the tables

The repository ships with the pre-computed CSV in `data/`, so the 
tables can be reproduced directly from the shipped data:

```bash
python plot_ablation_table_realworld.py
```

This will produce the table images in this directory.

### Regenerating the data from scratch

To regenerate the CSV from scratch, run:

```bash
python comparison_ablation_realworld.py
```

This produces both `data/results_ablation_realworld.csv` (long 
format, one row per simulation) and 
`data/summary_ablation_realworld.csv` (mean ± SEM). Once the CSV 
is in place:

```bash
python plot_ablation_table_realworld.py
```


## Configuration

Key parameters can be adjusted near the top of 
`comparison_ablation_realworld.py`:

- `LAMBDA_VALUES = [1, 10, 20]` — the three evasion-factor levels
- `SIMULATION_TIMES = 50` — independent simulations per condition
- `SEED = 14` — global RNG seed for the experiment driver
- `N_WORKERS` — multiprocessing pool size, capped automatically 
  at $60$ on Windows (due to the 63-handle limit of 
  `WaitForMultipleObjects`); set to $128$ on Linux / macOS.
- `MODEL_PATHS` — the four variant identifiers mapped to their 
  `model.pt` files. Edit if you re-train a variant under a 
  different seed/run name.

For the rendering script, no configuration is normally needed; 
`CSV_PATH` and `SAVE_DIR` are resolved automatically.


## Architectural notes

### Why model weights live in two places

The full MobileIsolator weights are shared by **every** 
experiment in this repository (Figs 3–6 and all SM figures that 
use the trained policy), so they live in the repository-level 
`train/` directory to avoid duplication. The three ablation 
variants, in contrast, are only used by this Supplementary Tables 
1–3 study, so they live alongside the experiment that consumes 
them. The script resolves both locations automatically; you do 
not need to copy or move any weights.

### Why the script imports from fig6

This ablation experiment evaluates the same six real-world 
scenarios as main-text Fig. 6, with the **same target-selection 
logic** for each scenario. Rather than re-implement that logic 
here, the script imports each scenario's data script from `fig6/` 
as a Python module and calls its `build_*_network()` (or, for 
COVID-19, `load_network()` + `generate_cluster_targets()`) 
function. This guarantees that the ablation study and Fig. 6 see 
identical inputs.
