# Supplementary Figure 6: Target-ratio generalization

This directory contains the scripts and data required to reproduce 
**Supplementary Figure 6** of the paper *Containment of Escaping 
Targets in Complex Networks*.

Supplementary Figure 6 tests how the four methods scale with the 
fraction $|T|/|V|$ of nodes designated as targets. MobileIsolator 
is trained with target ratios sampled from $[0.01, 0.10]$; this 
figure evaluates all methods on a $50\%$ wider sweep, 
$|T|/|V| \in \{0.01, 0.02, \ldots, 0.15\}$, on the four synthetic 
topology–target regimes at the fixed evasion factor $\lambda = 
20$. The figure quantifies the gap $\Delta P_C$ between 
MobileIsolator and Adaptive TIA as the target ratio increases 
beyond the training range.


## Files

### Scripts

| Script | Role |
|--------|------|
| `comparison_method_target_ratio.py` | Generates the data: sweeps target ratio across the four regimes at $\lambda = 20$ on $N = 1024$ networks, averaged over $100$ independent simulations per condition. Imports shared utilities from `../../fig3/comparison_method.py`. |
| `draw_target_ratio.py` | Renders Supplementary Figure 6 from the CSV. |

### Data (`data/`)

| File | Description | Source |
|------|-------------|--------|
| `results_target_ratio_n1024_lambda20.0_generalization.csv` | Aggregate $P_C$ and ANC statistics (means and SEMs) for each $(\textrm{target ratio}, \textrm{regime}, \textrm{method})$ tuple. | Generated by `comparison_method_target_ratio.py` |


## Dependencies

`comparison_method_target_ratio.py` imports shared utilities from 
the Fig. 3 directory:

```python
from comparison_method import (...)              # ../../fig3/
```

The MobileIsolator agent class is loaded transitively from the 
repository-level `train/` directory:

```python
from ppo_gnn_moving_gat_dualstream_final import DualStreamAgent  # ../../train/
```

The pre-trained policy weights are loaded from:

```
../../train/gnn_ppo_dual_stream_n50-100__seed42__1769653775/model.pt
```

Path resolution is automatic; the script locates `fig3/` and 
`train/` relative to its own location (two levels up, since this 
directory is a sub-directory of `fig_sm/`). The plotting script 
`draw_target_ratio.py` has no such dependency — it only reads 
the CSV from `data/`.


## Reproducing the figure

The repository ships with the pre-computed CSV in `data/`, so the 
figure can be reproduced directly from the shipped data:

```bash
python draw_target_ratio.py
```

This will produce `draw_target_ratio.pdf` and `.png` in this 
directory.

### Regenerating the data from scratch

To regenerate the CSV from scratch, run:

```bash
python comparison_method_target_ratio.py
```

This produces 
`data/results_target_ratio_n1024_lambda20.0_generalization.csv`. 
Once the CSV is in place:

```bash
python draw_target_ratio.py
```


## Configuration

Key parameters can be adjusted inside the 
`if __name__ == "__main__"` block of 
`comparison_method_target_ratio.py`:

- `N_NODES = 1024` — synthetic network size
- `LAMBDA_EFF = 20.0` — fixed evasion factor
- `TARGET_RATIO_VALUES` — the target-ratio sweep grid (default 
  $0.01$ to $0.15$ in steps of $0.01$, i.e. $50\%$ beyond the 
  training range)
- `SIMULATION_TIMES = 100` — independent simulations per condition
- `N_WORKERS`, `N_WORKERS_RL` — multiprocessing pool sizes. 
  Automatically capped at $60$ on Windows (due to the 63-handle 
  limit of `WaitForMultipleObjects`); set to $63$ on Linux / 
  macOS.

For the rendering script, no configuration is normally needed; 
`CSV_PATH` and `SAVE_DIR` are resolved automatically.
