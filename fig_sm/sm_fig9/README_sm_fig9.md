# Supplementary Figure 9: Fully adaptive vs. batch adaptive strategies

This directory contains the scripts and data required to reproduce 
**Supplementary Figure 9** of the paper *Containment of Escaping 
Targets in Complex Networks*.

Supplementary Figure 9 compares two decision-update regimes for 
all four methods on the four synthetic topology–target settings:

- **Fully adaptive** — the policy (or heuristic ranking) is 
  re-evaluated after every removal, so each decision uses the 
  most recent target positions and graph state.
- **Batch adaptive** — the policy is re-evaluated only once per 
  movement step; within a step, the removal budget for that step 
  is executed without intermediate re-evaluation.

The figure plots $P_C$ and ANC as a function of the evasion factor 
$\lambda$ for each $(\textrm{method}, \textrm{strategy}, 
\textrm{regime})$ combination, laid out as $2$ rows (PC / ANC) 
$\times$ $4$ columns (regimes) with solid lines for Fully and 
dashed lines for Batch.


## Files

### Scripts

| Script | Role |
|--------|------|
| `comparison_adaptive_method.py` | Generates the data: runs each method under both Fully and Batch adaptive strategies, on the four regimes, across the $\lambda$ sweep, averaged over $100$ independent simulations per condition. The script is self-contained and defines its own GNN agent wrapper; it does not import from `fig3/comparison_method.py`. |
| `draw_comparison_adaptive.py` | Renders Supplementary Figure 9 from the CSV. |

### Data (`data/`)

| File | Description | Source |
|------|-------------|--------|
| `comparison_adaptive_results.csv` | Aggregate $P_C$ and ANC statistics (means and SEMs) for each $(\textrm{regime}, \textrm{base\_method}, \textrm{strategy}, \lambda)$ tuple. | Generated by `comparison_adaptive_method.py` |


## Dependencies

`comparison_adaptive_method.py` loads the MobileIsolator agent 
class from the repository-level `train/` directory:

```python
from ppo_gnn_moving_gat_dualstream_final import DualStreamAgent
```

The pre-trained policy weights are loaded from:

```
../../train/gnn_ppo_dual_stream_n50-100__seed42__1769653775/model.pt
```

Path resolution is automatic; the script locates `train/` 
relative to its own location (two levels up, since this directory 
is a sub-directory of `fig_sm/`). Unlike most other comparison 
scripts in this repository, this one does **not** depend on 
`fig3/comparison_method.py` — all required helpers (heuristic 
methods, simulators, graph generators) are defined inline.

The plotting script `draw_comparison_adaptive.py` has no such 
dependency — it only reads the CSV from `data/`.


## Reproducing the figure

The repository ships with the pre-computed CSV in `data/`, so the 
figure can be reproduced directly from the shipped data:

```bash
python draw_comparison_adaptive.py
```

This will produce `comparison_adaptive.pdf` and `.png` in this 
directory.

### Regenerating the data from scratch

To regenerate the CSV from scratch, run:

```bash
python comparison_adaptive_method.py
```

This produces `data/comparison_adaptive_results.csv`. Once the 
CSV is in place:

```bash
python draw_comparison_adaptive.py
```


## Configuration

Key parameters can be adjusted near the top of 
`comparison_adaptive_method.py`:

- `N_NODES = 1024` — synthetic network size
- `TARGET_RATIO = 0.05` — fraction of nodes designated as targets
- `LAMBDA_EFF_VALUES = [1] + list(range(2, 22, 2))` — evasion 
  factor sweep grid
- `SIMULATION_TIMES = 100` — independent simulations per condition
- `N_WORKERS` — multiprocessing pool size. Scales automatically 
  with the host (`max(1, mp.cpu_count() - 1)`), and is 
  additionally capped at $60$ on Windows due to the 63-handle 
  limit of `WaitForMultipleObjects`.

For the rendering script, no configuration is normally needed; 
`CSV_PATH` and `SAVE_DIR` are resolved automatically.
